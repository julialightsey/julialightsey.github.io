[sql]

I spent several years developing a b-tree database in ANSI C during the late 1980's, using statistical methods to build an expert system on top of that database for decision support for the Office of Surface Mining. Later, I spent years writing Java and C# code against both SQL Server and Oracle Database, and became quite enamored of object oriented (OO) programming and design.

I found that I became more involved with SQL programming as a language, and with SQL Server as a platform. I remain impressed with the simplicity of the language relative to its power and potential sophistication.

I believe that most of the bad SQL queries in the world are the result of bad schema design. Joe Celko, SQL Programming Style.

But SQL is a language that allows the uninformed to build incredibly bad architectures. Designs that run well in boardroom presentations fail catastrophically when put under production load as data builds up.

[engineering]

Some of the oldest living things on this planet are trees. There is no mistaking either the beauty of trees or their success as a design.

Trees are designed using deterministic chaos, each tree, each leaf being a fractal based on the core function, the dna. Each tree is completely unique while being perfectly matched to its surroundings and requiring no maintenance. Improvement comes gradually and as required due to constant change and growth.

Science can amuse and fascinate us all, but it is engineering that changes the world. Isaac Asimov.

We can have no higher goal as engineers than to design and build artifacts that mimic the beauty of trees.
