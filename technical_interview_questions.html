[technical_interview_questions]

  1. [utility_construct]
    1. do you have a utility database or construct?
    2. If so, how do you make utilities available to all databases on an instance and to all instances in a SQL landscape?
  2. [meta_data_construct]
    1. do you have a meta data construct available?
    2. How is it implemented?
  3. [testing]
    1. how do you unit test SQL code?
    2. do you have any persistent programmatic tests? How have you approached mock data?
    3. how do you regression test when you promote to production?
    4. what does your test and development landscape look like?
    5. how do you test for performance?
    6. are your performance tests implicit or explicit; can you test for them programmatically (explicit) or are they "gut feel" based?
  4. [development_standards_best_practice_review]
    1. do you have stated development practices including naming convention and coding standards?
    2. if so, what are they?
    3. how do you measure objects for complexity?
    4. how do you maintain, adhere to, and review for development standards?
    5. how do you define best practice within the context of development standards?
    6. do you use a programmatic code formatting tool?
  5. [beauracracy]
    1. what is required to implement an object in production?
    2. how much time is required routinely for administrative tasks and meetings? how onerous are they?
  6. [documentation]
    1. how do you document schema objects and both structural and descriptive meta data?
    2. do you use extended properties for documentation?
